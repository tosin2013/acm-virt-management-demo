|===
| Module | Duration | Focus

| Overview and Environment Walkthrough
| 5 min
| Verify clusters, ACM console navigation

| 10 min
| Apply policies, verify compliance, trigger remediation

| Module 2: Fleet Observability
| 10 min
| Grafana dashboards, fleet-wide and per-VM metrics

| Module 3: Application Topology Views
| 8 min
| Deploy VM app via GitOps, inspect topology, remote logs

| Module 4: Deploy Without Cluster-Admin
| 8 min
| Bind subscription-admin, deploy via Git Subscription

| Module 5: Fine-Grained ACM Permissions
| 8 min
| Apply ClusterPermission, demonstrate scoped VM access

| Module 6: Eradicate Cluster Destruction
| 6 min
| Apply non-destructive role, attempt and fail to delete cluster

| *Total*
| *~55 min*
|

== Key API Groups

[cols="1,2", options="header"]
|===
| API Group | Purpose in This Demo

| `policy.open-cluster-management.io`
| VM governance policies, placement bindings

| `cluster.open-cluster-management.io`
| Managed cluster lifecycle, cluster sets

| `rbac.open-cluster-management.io`
| ClusterPermission API for fine-grained access

| `hive.openshift.io`
| Cluster provisioning lifecycle (protected from deletion)

| `kubevirt.io`
| VirtualMachine and VirtualMachineInstance resources

| `subresources.kubevirt.io`
| VM start/stop/restart/pause/console operations

| `monitoring.coreos.com`
| PrometheusRule for VM right-sizing metrics
|===

== Troubleshooting

[cols="1,2", options="header"]
|===
| Issue | Resolution

| ACM console not loading
| Verify the `multiclusterhub` pod is running: `oc get pods -n open-cluster-management`

| Managed cluster shows "Unknown" status
| Check the klusterlet agent: `oc get pods -n open-cluster-management-agent` on the managed cluster

| Grafana dashboards empty
| Verify observability is enabled: `oc get multiclusterobservability -o yaml`

| Policies show "NonCompliant" unexpectedly
| Check policy controller logs: `oc logs -n open-cluster-management deploy/governance-policy-propagator`

| VM operations denied
| Verify ClusterPermission was applied: `oc get clusterpermissions -n <managed-cluster-name>`
|===
